# Passport Flow

## Overview

The Reproducibility Passport is the final output generated by ReproForge Sentinel.

It is created by combining the results from claim parsing, evidence collection, repository risk analysis, and scoring into a single explainable report.

The Passport is generated only after all verification stages have completed.

---

# Verification Flow

```
                User / Judge
                     │
                     ▼
              Submit Claim
                     │
                     ▼
             Claim Parser
        (Extract technical claims)
                     │
                     ▼
          Evidence Collector
     (Collect files, logs, metadata,
      documentation and outputs)
                     │
                     ▼
          Evidence Validator
  (Check evidence quality and validity)
                     │
                     ▼
            Risk Scanner
 (Detect risky commands, secrets,
 dependencies, permissions, etc.)
                     │
                     ▼
           Scoring Engine
 (Calculate risk, reproducibility,
 credibility and evidence coverage)
                     │
                     ▼
        Passport Generator
 (Combine all verification results)
                     │
                     ▼
      Reproducibility Passport
                     │
                     ▼
          Dashboard / API Output
```

---

# Step-by-Step Process

## Step 1 – Claim Submission

The user submits a technical claim.

Examples:

- "This repository achieves 95% accuracy."
- "This agent safely edits configuration files."
- "This model supports AMD ROCm."

---

## Step 2 – Claim Parsing

The Claim Parser identifies:

- Main technical claim
- Repository or workflow
- Expected outcome
- Verification targets

Example:

Claim:

> "This repository achieves 95% accuracy."

Extracted Information:

- Performance claim
- Expected accuracy
- Repository reference

---

## Step 3 – Evidence Collection

The system searches for supporting artifacts.

Examples:

- README
- Dataset
- Evaluation script
- requirements.txt
- Dockerfile
- Model weights
- Execution logs

Each discovered artifact becomes an Evidence Item.

---

## Step 4 – Evidence Validation

Each evidence item is checked for:

- Traceability
- Completeness
- Relevance
- Verification status

Evidence receives one of the following statuses:

- Verified
- Partially Verified
- Missing
- Failed
- Warning

---

## Step 5 – Risk Analysis

The Risk Scanner analyzes the repository or workflow.

Checks may include:

- Dangerous shell commands
- Privilege escalation
- Network downloads
- Exposed secrets
- Suspicious dependencies
- Missing evaluation assets

Detected findings become Risk Indicators.

---

## Step 6 – Score Calculation

The Scoring Engine combines collected evidence and detected risks.

Generated metrics include:

- Risk Score
- Reproducibility Score
- Credibility Score
- Evidence Coverage

These scores summarize the overall verification result.

---

## Step 7 – Passport Generation

The Passport Generator combines:

- Original claim
- Verification summary
- Evidence items
- Risk indicators
- Scores
- Recommendations
- Final verdict

into a single Reproducibility Passport.

---

## Step 8 – Result Presentation

The completed Passport is returned to the user through:

- Dashboard
- API
- Export (future)

The Passport provides an explainable summary of the verification process rather than raw logs.

---

# Final Passport

The generated Passport includes:

- Passport Information
- Technical Claim
- Verification Summary
- Evidence Summary
- Risk Assessment
- Reproducibility Assessment
- Missing Evidence
- Recommendations
- Verification Logs
- Final Verdict

---

# Design Principles

The Passport Flow should ensure that every verdict is:

- Evidence-driven
- Explainable
- Traceable
- Reproducible
- Easy to understand

The Passport is the final representation of the verification process. Every score, recommendation, and verdict shown in the Passport must be supported by collected evidence.
